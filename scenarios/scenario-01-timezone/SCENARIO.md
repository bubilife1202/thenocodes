# Scenario #1 — 깨진 미팅 날짜 포매터 고치기

## 상황 (Situation)

더노코즈 밋업 페이지에 붙은 `formatMeetupDate()` 유틸이 최근 QA에서
"날짜가 이상하다"는 제보를 받았다. 테스트 3개 중 1개가 실패한다.

AI와 함께 원인을 찾고 고쳐라. 단, 통과하는 테스트 2개는 버그를 가리고 있다.

## 시작 방법 (How to start)

현재는 더노코즈 사이트에서 ZIP으로 배포한다. GitHub 공개 리포는 별도 Gate 2에서 열린다.

1. 스타터 ZIP을 내려받고 압축을 푼다.
2. `npm ci`로 의존성을 설치한다.
3. `npm test`를 실행한다. 시작 상태에서는 3개 중 1개가 의도적으로 실패해야 한다.
4. 실패 테스트를 통과시키는 최소 수정을 적용한다.
5. 통과 중인 테스트가 가리고 있는 숨은 경계 버그를 찾아 회귀 테스트를 추가한다.
6. `PROMPT_LOG_TEMPLATE.md`를 복사해 AI 협업, 검증, 수정 과정을 기록한다.

## 당신의 미션 (Deliverables)

1. **실패하는 테스트를 통과시키는 최소 수정.** 나머지 테스트가 깨지면 안 된다.
2. **숨어있는 버그를 찾아라.** 통과하는 테스트 2개는 버그를 가리고 있다.
   발견하지 못하면 다음 QA에서 고객이 먼저 발견할 것이다.
3. **PR diff 또는 패치 + 1장 분량 회고록 + `PROMPT_LOG_TEMPLATE.md` 기반 프롬프트 로그.**
   AI에게 무엇을 물었는지, AI가 무엇을 틀렸는지, 어떻게 발견했는지, 최종적으로 무엇을 바꿨는지 기록한다.

## 제출 방법 (How to submit)

Gate 1에서는 수동 제출이다.

1. `git diff`를 저장한 `.patch`, GitHub PR 링크(본인 fork를 사용한 경우), 회고록, 프롬프트 로그를 함께 준비한다.
2. 제출/문의 주소는 `hello@thenocodes.org` 이다.
3. 채점 결과는 이메일로 회신된다. 아직 로그인, 결제 연동, 제출 대시보드는 없다.

## 프롬프트 로그

`PROMPT_LOG_TEMPLATE.md` 를 복사해서 채운다.
어떤 AI 도구를 쓰든 통일된 포맷으로 기록한다.
(Claude Code, Cursor, ChatGPT, Gemini, Aider 모두 가능.)

## 채점 기준

`../RUBRIC.md` 를 참조. 3개 축 x 4 레벨.

| 축 | 무엇을 본다 |
|---|---|
| **Collaboration** | 태스크를 잘게 쪼갠 프롬프트 vs 모든 것을 한 번에 던지는 프롬프트 |
| **Verification** | AI 답을 그대로 수락했나 vs 독립 신호로 검증했나 |
| **Improvement** | 보이는 버그만 고쳤나 vs 숨은 버그까지 커버하는 회귀 테스트 추가했나 |

## 참고

현재 배포 방식은 더노코즈 사이트의 ZIP 스타터 패키지다.
공개 GitHub 리포는 아직 열려 있지 않으며, Gate 2에서 실제 소유자와 경로가 확정되면 링크를 교체한다.
운영자용 `SOLUTIONS.md`는 스타터 패키지에 포함되지 않는다.
