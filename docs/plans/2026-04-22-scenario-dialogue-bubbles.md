# /scenarios 말풍선 UI 기획

## 목적

`/scenarios`는 "AI와 함께 풀었다"는 주장만으로는 약하다. 사용자가 실제로 무엇을 사는지 보려면 최종 코드보다 먼저 대화의 움직임이 보여야 한다.

말풍선 UI는 학습자와 AI의 대화를 장식처럼 보여주는 요소가 아니라, 더노코즈가 채점하는 세 가지 능력(협업, 검증, 개선)을 한눈에 증명하는 표본이다.

## 핵심 결정

- 말풍선은 라이브 채팅 기능이 아니다.
- 말풍선은 실제 풀이 로그를 압축한 정적 발췌다.
- 랜딩에는 짧은 4개 버블만 둔다.
- 전체 대화와 채점 근거는 `/scenarios/walkthrough`에서 확장한다.
- 모든 버블 묶음은 AI가 틀리거나 불완전했던 순간을 포함해야 한다. 그 순간을 학습자가 어떻게 검증하고 다음 프롬프트에 반영했는지가 상품의 핵심이다.

## 랜딩 구성

위치: `/scenarios`의 "첫 번째 시나리오"와 "채점은 어떻게 하나요" 사이.

제목: `실제 풀이 대화는 이렇게 남깁니다`

보조문:

> 정답만 제출하지 않습니다. 어떤 맥락을 줬고, AI가 어디서 빗나갔고, 그걸 어떻게 검증해서 다시 지시했는지를 함께 봅니다.

버블 4개:

1. Learner
   `formatMeetupDate 테스트가 하나 실패합니다. 먼저 실패 원인을 설명하고, 코드 수정은 아직 하지 마세요.`

2. AI
   `KST로 날짜가 넘어가는데 요일은 UTC 기준으로 읽고 있어 화요일로 남습니다.`

3. Verification
   `테스트 2개가 통과하는데, rawHour === 24 경계값은 아직 검증하지 않았습니다. 자정 직후 케이스를 하나 더 추가해봅니다.`

4. Revision
   `요일만 고치지 말고 날짜 advance 조건도 >= 24로 바꿔야 합니다. 이 경계값 회귀 테스트까지 포함해 최소 diff로 다시 제안해주세요.`

CTA: `전체 워크스루에서 채점 근거 보기 -> /scenarios/walkthrough`

## 시각 규칙

- Learner: 왼쪽 정렬, 흰 배경, 얇은 테두리.
- AI: 오른쪽 정렬, 옅은 teal 배경.
- Verification: 왼쪽 정렬, amber 계열 얇은 테두리.
- Revision: 왼쪽 정렬, green 계열 얇은 테두리.
- 모바일에서는 좌우 폭 차이를 줄이고, role 라벨로 구분한다.
- 말풍선 안 텍스트는 2줄 내외로 압축한다. 긴 로그는 walkthrough로 보낸다.

## Walkthrough 확장

`/scenarios/walkthrough`에는 전체 대화를 그대로 붙이지 않는다. 대신 3개 축별로 버블 묶음을 나눈다.

- Collaboration 묶음: 좋은 첫 프롬프트, 제약, 소스 첨부.
- Verification 묶음: AI 답변 수락 전 테스트/직접 독해/엣지케이스 확인.
- Improvement 묶음: 검증 결과를 다음 프롬프트와 최종 diff에 반영.

각 묶음 아래에 `왜 L3이고 L2/L4가 아닌가`를 현재 walkthrough와 연결한다.

## 구현 범위

1차 구현:

- `src/app/scenarios/page.tsx`에 정적 말풍선 섹션 추가.
- `src/app/scenarios/walkthrough/page.tsx`에 축별 버블 묶음 추가.
- 재사용 컴포넌트는 필요할 때만 만든다. 처음에는 페이지 안 작은 로컬 배열 + map으로 충분하다.

하지 않는 것:

- 실시간 AI 채팅 UI.
- 사용자의 프롬프트 업로드/저장.
- 채점 대시보드.
- LLM 자동 채점.

## 검증

- `npm run build`
- `curl -s -o /tmp/scenarios.html -w '%{http_code}' https://thenocodes.org/scenarios`가 200.
- 랜딩 HTML에 `실제 풀이 대화는 이렇게 남깁니다`, `Verification`, `Revision`이 포함됨.
- 모바일 폭에서 말풍선 텍스트가 카드 밖으로 넘치지 않음.
